## What is tsim?

tsim is a quantum circuit simulator designed for fast sampling from Clifford+T circuits with Pauli noise. It mirrors the [Stim](https://github.com/quantumlib/Stim) API while extending support to non-Clifford gates including T, T_DAG, rotation gates (R_X, R_Y, R_Z), and the general single-qubit unitary U3.

The underlying algorithm combines stabilizer rank decomposition with ZX-calculus simplification rules, enabling efficient simulation of hundreds of qubits as long as circuits don't contain too many non-Clifford instructions.

## Use Cases

tsim is particularly well-suited for:

- **Quantum Error Correction Research**: Simulate error correction circuits with realistic noise models
- **Magic State Distillation**: Analyze and optimize T-state preparation protocols
- **Clifford+T Circuit Analysis**: Evaluate quantum algorithms that mix Clifford and non-Clifford operations
- **Detector Sampling**: Generate detector samples for quantum error correction experiments
- **Non-Deterministic Measurements**: Handle circuits where detectors and observables have inherent randomness

## How It Works

tsim compiles quantum circuits into measurement or detector samplers following the approach described in [Sutcliffe and Kissinger (2025)](https://arxiv.org/abs/2403.06777). These compiled samplers:

1. **Decompose** the circuit using stabilizer rank decomposition
2. **Simplify** the resulting ZX-graph using rewrite rules
3. **Compile** into efficient JAX code for CPU or GPU execution
4. **Sample** using autoregressive sampling over the decomposition

This architecture allows tsim to handle circuits with non-Clifford gates while maintaining high performance through GPU acceleration and careful memory management.

## Important Differences from Stim

<Warning>
tsim supports **non-deterministic detectors and observables**. This means tsim simulates actual detector samples, whereas Stim reports detection flips (samples XORed with a noiseless reference).

For example, a circuit with `X 0; M 0; DETECTOR rec[-1]` will produce `True` values in tsim but `False` values in Stim.
</Warning>
