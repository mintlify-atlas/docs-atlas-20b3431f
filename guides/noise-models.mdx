tsim supports comprehensive Pauli noise modeling, including all Stim noise channels plus the ability to generate detector error models for integration with quantum error correction decoders.

## Overview

tsim implements noise as probability distributions over Pauli errors. The noise channels are compiled into efficient samplers that use geometric-skip sampling optimized for low-noise regimes.

### Available Noise Channels

- **Single-qubit depolarizing**: `DEPOLARIZE1(p)`
- **Two-qubit depolarizing**: `DEPOLARIZE2(p)`
- **Single-qubit Pauli channel**: `PAULI_CHANNEL_1(px, py, pz)`
- **Two-qubit Pauli channel**: `PAULI_CHANNEL_2(...)`
- **Single-Pauli errors**: `X_ERROR(p)`, `Y_ERROR(p)`, `Z_ERROR(p)`
- **Correlated errors**: `CORRELATED_ERROR(...)`

## Depolarizing Noise

### Single-Qubit Depolarizing

The single-qubit depolarizing channel applies X, Y, or Z errors with equal probability p/3 each:

```python
import tsim

c = tsim.Circuit(
    """
    RX 0
    H 0
    DEPOLARIZE1(0.01) 0  # 1% total error rate
    M 0
    """
)
sampler = c.compile_sampler()
samples = sampler.sample(shots=1000)
```

The `DEPOLARIZE1(p)` channel applies:
- No error with probability (1 - p)
- X error with probability p/3
- Y error with probability p/3  
- Z error with probability p/3

### Two-Qubit Depolarizing

The two-qubit depolarizing channel applies each of the 15 non-identity two-qubit Pauli operators with equal probability p/15:

```python
c = tsim.Circuit(
    """
    H 0
    CNOT 0 1
    DEPOLARIZE2(0.01) 0 1  # 1% total error rate
    M 0 1
    """
)
```

This is equivalent to `PAULI_CHANNEL_2` with all 15 error probabilities set to p/15.

## Pauli Channels

### Single-Qubit Pauli Channel

Specify individual probabilities for X, Y, and Z errors:

```python
import tsim

c = tsim.Circuit(
    """
    RX 0
    R 1
    T 0
    PAULI_CHANNEL_1(0.1, 0.1, 0.2) 0 1  # px=0.1, py=0.1, pz=0.2
    H 0
    CNOT 0 1
    M 0 1
    """
)
sampler = c.compile_sampler()
samples = sampler.sample(shots=100)
```

The channel applies:
- No error with probability (1 - px - py - pz)
- X error with probability px
- Y error with probability py
- Z error with probability pz

Implementation: `src/tsim/noise/channels.py:35-41`

### Two-Qubit Pauli Channel

Specify individual probabilities for all 15 two-qubit Pauli errors:

```python
c = tsim.Circuit(
    """
    CNOT 0 1
    PAULI_CHANNEL_2(0.01, 0.01, 0.01,  # pix, piy, piz
                    0.01, 0.01, 0.01, 0.01,  # pxi, pxx, pxy, pxz
                    0.01, 0.01, 0.01, 0.01,  # pyi, pyx, pyy, pyz
                    0.01, 0.01, 0.01, 0.01   # pzi, pzx, pzy, pzz
                   ) 0 1
    M 0 1
    """
)
```

The 15 parameters correspond to the two-qubit Pauli basis (excluding II):
- `pix`, `piy`, `piz` - First qubit I, second qubit X/Y/Z
- `pxi`, `pxx`, `pxy`, `pxz` - First qubit X, second qubit I/X/Y/Z
- `pyi`, `pyx`, `pyy`, `pyz` - First qubit Y, second qubit I/X/Y/Z
- `pzi`, `pzx`, `pzy`, `pzz` - First qubit Z, second qubit I/X/Y/Z

Implementation: `src/tsim/noise/channels.py:43-100`

## Single-Pauli Errors

Apply specific Pauli errors with given probabilities:

```python
import tsim

c = tsim.Circuit(
    """
    H 0
    X_ERROR(0.01) 0  # 1% X error
    Z_ERROR(0.02) 0  # 2% Z error
    M 0
    """
)
```

Available single-Pauli error channels:
- `X_ERROR(p)` - Apply X with probability p
- `Y_ERROR(p)` - Apply Y with probability p
- `Z_ERROR(p)` - Apply Z with probability p

Implementation: `src/tsim/noise/channels.py:588-608`

## Correlated Errors

Correlated errors allow multiple qubits to experience coordinated Pauli errors:

```python
import tsim

c = tsim.Circuit(
    """
    H 0
    CNOT 0 1 2
    CORRELATED_ERROR(0.01) X0 X1 X2  # Correlated X errors on qubits 0,1,2
    M 0 1 2
    """
)
```

### Conditional Correlated Errors

Use `ELSE_CORRELATED_ERROR` to define mutually exclusive error mechanisms:

```python
c = tsim.Circuit(
    """
    CNOT 0 1
    CORRELATED_ERROR(0.01) X0 X1  # 1% chance of X0*X1
    ELSE_CORRELATED_ERROR(0.005) Z0 Z1  # If no X0*X1, 0.5% chance of Z0*Z1
    M 0 1
    """
)
```

The conditional probability structure is implemented in `src/tsim/noise/channels.py:103-131`.

## Measurement Noise

Add noise to measurement operations by specifying an error probability:

```python
import tsim

c = tsim.Circuit(
    """
    H 0
    M(0.01) 0  # 1% measurement error (bit flip)
    """
)
```

Measurement noise is implemented as an X error applied before the ideal measurement.

## Detector Error Models

Generate detector error models for integration with QEC decoders like PyMatching:

```python
import tsim

c = tsim.Circuit(
    """
    RX 0
    R 1
    CNOT 0 1
    X_ERROR(0.01) 0 1
    M 0 1
    DETECTOR rec[-1] rec[-2]
    """
)

# Generate detector error model
dem = c.detector_error_model()
print(dem)
```

### Non-Deterministic Observables

tsim supports non-deterministic observables when generating detector error models:

```python
dem = c.detector_error_model(
    allow_non_deterministic_observables=True
)
```

This is achieved by temporarily converting observables to detectors, generating the DEM with gauge detectors allowed, then converting back. See implementation: `src/tsim/noise/dem.py:8-217`

## Advanced: Channel Sampling Implementation

### Geometric-Skip Sampling

tsim uses an optimized sampling algorithm for low-noise regimes. The `ChannelSampler` class implements geometric-skip sampling that's highly efficient when error probabilities are small.

Key features:
- **Channel simplification**: Merges channels with identical signatures
- **Subset absorption**: Combines channels when one's signatures are a subset of another's
- **Basis transformation**: Efficiently transforms error bits to detector bits using GF(2) linear algebra

Implementation: `src/tsim/noise/channels.py:380-533`

### Channel Simplification

Before sampling, channels are automatically simplified:

1. **Null bit reduction**: Remove error bits that don't affect any output
2. **Normalization**: Sort column signatures for consistent comparison
3. **Merging**: Combine channels with identical signatures via XOR convolution
4. **Absorption**: Expand and merge channels where one is a subset of another

Implementation: `src/tsim/noise/channels.py:358-377`

## Practical Examples

### Example 1: Noisy Bell State

```python
import tsim

c = tsim.Circuit(
    """
    # Prepare Bell state
    H 0
    CNOT 0 1
    
    # Add various noise sources
    DEPOLARIZE1(0.01) 0 1
    DEPOLARIZE2(0.005) 0 1
    
    # Measure with noise
    M(0.001) 0 1
    """
)
sampler = c.compile_sampler()
samples = sampler.sample(shots=10000)
```

### Example 2: Repetition Code with Detectors

```python
import tsim

c = tsim.Circuit(
    """
    # Encode
    RX 0
    CNOT 0 1
    CNOT 0 2
    
    # Noise
    PAULI_CHANNEL_1(0.01, 0.01, 0.01) 0 1 2
    
    # Measure and detect
    M 0 1 2
    DETECTOR rec[-3] rec[-2]
    DETECTOR rec[-2] rec[-1]
    """
)
detector_sampler = c.compile_detector_sampler()
samples = detector_sampler.sample(shots=1000)
```

### Example 3: T Gate with Noise

```python
import tsim

c = tsim.Circuit(
    """
    RX 0
    H 0
    
    # Noisy T gate
    T 0
    DEPOLARIZE1(0.001) 0
    
    # More gates
    H 0
    M 0
    """
)
sampler = c.compile_sampler()
samples = sampler.sample(shots=5000)
```

## Performance Tips

### Low-Noise Optimization

tsim's geometric-skip sampler is optimized for low error rates (p < 0.1). For higher error rates, consider:
- Using fewer shot counts per call
- Simplifying the circuit
- Reducing the number of independent noise channels

### Channel Count

The number of independent noise channels affects compilation time. Channels are automatically simplified, but you can reduce overhead by:
- Using `DEPOLARIZE1` instead of multiple `PAULI_CHANNEL_1` when appropriate
- Grouping correlated errors when they represent the same physical mechanism
