The `tsim.core` module provides the fundamental building blocks for ZX graph construction, manipulation, and circuit parsing.

## Overview

This module contains:
- **graph**: ZX graph construction and manipulation
- **parse**: Stim circuit parsing to ZX graphs
- **instructions**: Gate implementations and graph representations
- **types**: Core data types for compilation pipeline
- **exact_scalar**: Exact arithmetic for ZX-calculus phases

## tsim.core.graph

ZX graph construction, manipulation, and preparation for sampling.

### Key Functions

#### `prepare_graph`

```python
def prepare_graph(circuit: Circuit, *, sample_detectors: bool) -> SamplingGraph
```

Prepare a circuit for compilation. This is the main entry point for graph preparation.

**Pipeline:**
1. Parse the stim circuit into a ZX graph
2. Build the sampling graph (compose with adjoint, add outputs)
3. Reduce the graph via `zx.full_reduce`
4. Transform error basis via Gaussian elimination (e → f)
5. Clear the scalar

**Parameters:**
- `circuit`: The quantum circuit to prepare
- `sample_detectors`: If True, prepare for detector sampling; if False, prepare for measurement sampling

**Returns:** `SamplingGraph` containing the reduced graph and error transformation

#### `transform_error_basis`

```python
def transform_error_basis(
    g: BaseGraph, 
    num_e: int | None = None
) -> tuple[BaseGraph, np.ndarray]
```

Transform phase variables from the original 'e' basis to a reduced 'f' basis.

Finds a linearly independent basis for phase variables and transforms them via Gaussian elimination. Original variables (e0, e1, ...) are mapped to a smaller set (f0, f1, ...).

**Parameters:**
- `g`: A ZX graph with phase variables attached to vertices
- `num_e`: Total number of e-variables (if provided, pads matrix to this size)

**Returns:** Tuple of (modified graph, binary transformation matrix)
- Matrix shape: `(num_f, num_e)` where entry `[i, j] = 1` means `f_i` depends on `e_j`
- Example: If row 0 is `[0, 1, 0, 1]`, then `f0 = e1 XOR e3`

## tsim.core.types

Core data types for the tsim compilation and sampling pipeline.

### SamplingGraph

```python
@dataclass(frozen=True)
class SamplingGraph:
    graph: BaseGraph
    error_transform: np.ndarray
    channel_probs: list[np.ndarray]
    num_outputs: int
    num_detectors: int
```

Result of the graph preparation phase for sampling.

Represents a circuit that has been:
1. Parsed from Stim format
2. Converted to a ZX graph
3. Doubled (composed with adjoint)
4. Reduced via `zx.full_reduce`
5. Had its error basis transformed (e → f)

**Attributes:**
- `graph`: The prepared ZX graph with f-parameters
- `error_transform`: Binary matrix `(num_f, num_e)` where `[i,j]=1` means f_i depends on e_j
- `channel_probs`: List of probability arrays for error channels
- `num_outputs`: Number of output vertices
- `num_detectors`: Number of detector vertices

## tsim.core.exact_scalar

Exact scalar arithmetic for ZX-calculus phase computations.

### Overview

Implements exact arithmetic for complex numbers of the form:

```
(a + b·e^(iπ/4) + c·i + d·e^(-iπ/4)) × 2^power
```

This representation enables exact computation without floating-point errors.

---

### ExactScalarArray


Exact scalar array using dyadic representation.

Represents values of the form `(c₀ + c₁·ω + c₂·ω² + c₃·ω³) × 2^power` where `ω = e^(iπ/4)`.
